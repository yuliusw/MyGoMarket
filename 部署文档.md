# 部署文档

本文描述本地 Docker Compose 部署、全量重建和验证流程。所有可提交配置均已脱敏，真实凭据通过本地 `.env` 注入。

## 目录

主要部署文件在 `script/`：

```text
script/docker-compose.yaml        本地容器编排
script/.env.example               可提交的环境变量示例
script/config.yaml                容器内应用默认配置，已脱敏
script/init_better.sql            PostgreSQL 初始化脚本
script/redis.conf                 Redis 配置文件
script/loki-config.yaml           Loki 配置
script/rmq/broker.conf            RocketMQ Broker 配置
script/deploy-reset.sh            全量重建和验证脚本
```

## 环境变量

第一次部署前，可从示例生成本地 `.env`：

```bash
cp script/.env.example script/.env
```

本地真实密码只改 `script/.env`，不要提交 `.env`。

关键变量：

```env
POSTGRES_USER=rpa_app
POSTGRES_PASSWORD=change-me-db-password
POSTGRES_DB=RPA

MINIO_ROOT_USER=rpa_minio_access_key
MINIO_ROOT_PASSWORD=change-me-minio-secret-key

GF_SECURITY_ADMIN_USER=rpa_admin
GF_SECURITY_ADMIN_PASSWORD=change-me-grafana-admin-password
```

应用会通过 compose 注入：

```text
DATABASE_USER
DATABASE_PASSWORD
DATABASE_DBNAME
MINIO_ACCESSKEY
MINIO_SECRETKEY
```

因此不需要把真实 DB/MinIO 密码写入 `config.yaml`。

## 全量重建

如需删除旧容器和旧数据卷，重新初始化一套干净环境，直接运行：

```bash
bash script/deploy-reset.sh
```

该脚本会执行：

```text
1. go build -o script/main .
2. docker build -t rpa-app:v1.0 script
3. docker compose config 校验配置
4. docker compose down -v --remove-orphans 删除旧容器和项目数据卷
5. docker compose up -d 启动全量服务
6. 等待 PostgreSQL 和 Redis healthcheck
7. 验证 DB 权限种子、Redis、MinIO、RocketMQ、应用 HTTP 接口
```

如果存在 `script/.env`，脚本会自动加载并传给 compose；如果不存在，则使用 compose 中的脱敏默认值。

注意：`down -v` 会删除 compose 项目的数据卷，包括 PostgreSQL、Redis、MinIO、Grafana、Prometheus、Alloy 数据。需要保留数据时不要使用全量重建脚本。

## 常规启动

不删除数据，只启动现有环境：

```bash
cd script
docker compose up -d
```

查看状态：

```bash
docker compose ps -a
```

停止但保留数据卷：

```bash
cd script
docker compose down
```

## 重新构建应用镜像

修改 Go 代码或 `script/config.yaml` 后，需要重建应用镜像：

```bash
go build -o ./script/main .
docker build -t rpa-app:v1.0 ./script
cd script
docker compose up -d app
```

## 初始化验证

### 容器状态

```bash
cd script
docker compose ps -a
```

期望核心服务为 `Up`：

```text
RPA-app
RPA-pgdb
RPA-redis
RPA-minio
RPA-loki
RPA-grafana
rmqnamesrv
rmqbroker
rmqdashboard
RPA-prometheus
RPA-alloy
RPA-cadvisor
```

`RPA-minio-setup` 是一次性初始化容器，`Exited (0)` 是正常状态。

### DB 权限种子

```bash
docker exec RPA-pgdb sh -c 'psql -U "$POSTGRES_USER" -d "$POSTGRES_DB" -P pager=off -c "SELECT '\''users'\'' AS table_name, count(*) FROM users UNION ALL SELECT '\''groups'\'', count(*) FROM groups UNION ALL SELECT '\''roles'\'', count(*) FROM roles UNION ALL SELECT '\''permissions'\'', count(*) FROM permissions UNION ALL SELECT '\''role_permissions'\'', count(*) FROM role_permissions ORDER BY table_name;"'
```

干净初始化后应为：

```text
users             1
groups            1
roles             4
permissions       15
role_permissions  33
```

角色权限数量应为：

```text
superadmin  15
owner        9
developer    7
member       2
```

### Redis

```bash
docker exec RPA-redis redis-cli ping
```

期望输出：

```text
PONG
```

### MinIO

使用 `.env` 中的 MinIO 凭据检查 bucket：

```bash
cd script
docker run --rm --network script_default --entrypoint /bin/sh minio/mc -c 'mc alias set myminio http://minio:9000 rpa_minio_access_key change-me-minio-secret-key >/dev/null && mc ls myminio'
```

默认应存在：

```text
loki-bucket/
rpa-bucket/
```

### RocketMQ

```bash
docker exec rmqbroker /home/rocketmq/rocketmq-5.3.4/bin/mqadmin clusterList -n rmqnamesrv:9876
```

期望看到 `DefaultCluster`、`broker-a` 且 `ACTIVATED=true`。

### 应用 HTTP

```bash
curl -fsS 'http://localhost:12660/api/v1/market/apps?page=1&page_size=1'
```

干净库期望返回类似：

```json
{"data":[],"page":1,"page_size":1,"total":0}
```

## 常见问题

### App 启动后退出，日志显示 DB connection refused

原因通常是 PostgreSQL 容器启动了，但数据库还没 ready。当前 compose 已给 DB/Redis 增加 healthcheck，并让 app 等 `service_healthy` 后启动。

手动恢复：

```bash
cd script
docker compose up -d db redis app
```

### Redis 挂载配置失败

`script/redis.conf` 必须是文件，不是目录。当前仓库已提供 `script/redis.conf`。

### Loki 连不上 MinIO

Loki 使用 `-config.expand-env=true`，并从 compose 注入 `MINIO_ROOT_USER`、`MINIO_ROOT_PASSWORD`。如果修改了 `script/.env` 中 MinIO 凭据，重启 Loki：

```bash
cd script
docker compose up -d loki
```

### 修改 Go 代码后容器仍是旧逻辑

需要重建二进制和镜像：

```bash
go build -o ./script/main .
docker build -t rpa-app:v1.0 ./script
cd script
docker compose up -d app
```
