# RPA Market 管理后台用户文档

本文档面向产品、前端和测试人员，用于确认当前系统已经支持哪些管理功能，以及哪些全局后台能力还未实现。

## 当前定位

当前前端已经具备“应用市场工作台”能力，包括应用浏览、购买、钱包、订单查看。严格意义上的“全局管理后台”目前只完成了 RBAC 角色与权限管理的后端接口，尚未形成完整的全局后台页面。

因此当前系统可分为三类后台能力：

- 个人工作台：普通用户登录后可使用市场、钱包、订单功能。
- 开发者工作台：开发者可发布、更新、下架、删除自己发布的应用。
- 全局管理能力：具备 `role:manage` 权限的管理员可管理系统角色和权限绑定。

## 登录与会话

### 已支持

- 用户注册。
- 用户登录。
- 登录后后端写入 `auth_token` 和 `session_id` Cookie。
- 前端同时保存 JWT 到 `localStorage.auth_token`。
- 私有接口依赖 JWT 和 Session Cookie。
- 修改密码后会清理会话，需要重新登录。

### 页面入口

- `/login`：登录 / 注册。

### 注意事项

- 浏览器端建议通过同域代理访问后端，确保 Cookie 正常携带。
- 纯 API 客户端如果只传 Bearer Token、不带 `session_id` Cookie，当前可能无法通过认证。

## 应用市场工作台

### 已支持

- 查看已上架应用列表。
- 按名称搜索应用。
- 查看应用分类、标签、文件大小、价格字段。
- 下载应用文件。
- 一步购买应用。
- 购买成功后展示订单 ID、钱包流水 ID、订阅 ID。

### 页面入口

- `/market`

### 购买链路

用户点击“购买并发放”后，前端调用：

```text
POST /api/v1/orders/purchase
```

后端会完成：

```text
创建订单 -> 钱包扣款 -> 写钱包流水 -> 发放 subscriptions -> 订单置为 paid
```

成功后可看到：

- `order_id`：订单 ID。
- `tx_id`：钱包扣款流水 ID。
- `subscription_id`：订阅发放 ID。

### 当前限制

- 购买金额当前由前端提交。
- 如果应用 `metadata.price` 不存在，前端默认使用 `10.0000 COIN`。
- 后续应改成后端从 `app.metadata.price` 或独立价格表派生金额。

## 开发者应用管理

### 已支持

- 发布应用。
- 上传应用安装包。
- 支持 `.zip`、`.gz`、`.tgz` 文件。
- 更新应用基础信息。
- 下架应用。
- 删除应用。
- 下载热榜计数。

### 权限要求

- 发布应用需要 `app:create`。
- 编辑应用需要 `app:edit`。
- 下架应用需要 `app:offshelf`。
- 删除应用需要 `app:delete`。

### 安全约束

- 上传文件会校验扩展名和文件头类型。
- MinIO 对象路径由服务端生成，不使用用户原始文件名作为路径。
- 发布支持 `idempotency_key`，避免重复提交。
- 更新、下架、删除会校验应用所有者。

### 当前限制

- 还没有独立的“应用审核后台”。
- 全局管理员暂不能通过专门后台页面审核、强制下架或恢复任意应用。
- 前端当前页面偏开发者/市场混合工作台，不是完整运营后台。

## 钱包管理

### 已支持

- 查看当前用户钱包。
- 查看当前用户钱包流水。
- 测试充值。
- 钱包扣款。
- 钱包转账后端接口已支持。
- 订单支付会自动扣当前用户钱包。

### 页面入口

- `/wallet`

### 数据字段

- `wallet_id`：钱包 ID。
- `balance`：当前余额。
- `currency_code`：币种，当前默认 `COIN`。
- `status`：钱包状态。

### 流水字段

- `tx_id`：流水 ID。
- `tx_type`：流水类型，例如 `recharge`、`consume`、`transfer_in`、`transfer_out`。
- `amount`：变动金额，正数收入，负数支出。
- `balance_after`：交易后余额快照。
- `reference_id`：关联业务单据，订单支付时为 `order_id`。
- `idempotency_key`：幂等键。

### 并发安全

- 扣款、转账、订单支付都在数据库事务内执行。
- 核心写操作使用 `SELECT ... FOR UPDATE` 锁钱包行。
- 余额不足不会写流水。

### 当前限制

- 当前页面只展示当前用户钱包。
- 没有全局钱包审计后台。
- 没有管理员代充值、冻结钱包、解冻钱包页面。
- 后端钱包状态字段存在，但缺少管理端冻结/解冻接口。

## 订单与订阅发放

### 已支持

- 查看当前用户订单列表。
- 一步购买应用创建并支付订单。
- 单独创建订单后再支付的后端接口已支持。
- 支付成功自动发放 `subscriptions`。
- 订单记录关联钱包流水和订阅发放结果。

### 页面入口

- `/orders`

### 订单状态

- `pending`：待支付。
- `paid`：已支付并发放订阅。
- `cancelled`：已取消。

### 支付成功后的关联关系

```text
orders.tx_id = wallet_transactions.tx_id
orders.subscription_id = subscriptions.sub_id
wallet_transactions.reference_id = orders.order_id
subscriptions.source_order_id = orders.order_id
```

### 当前限制

- 当前页面只展示当前用户订单。
- 没有全局订单查询后台。
- 没有退款流程。
- 没有订单异常处理后台。
- 没有订阅撤销、续费、过期提醒页面。

## 群组与成员管理

### 已支持

- 创建群组。
- 查看我的群组。
- 查看群组详情和成员。
- 修改群组名称。
- 解散群组。
- 邀请成员。
- 移除成员。
- 修改成员角色。
- 主动退出群组。

### 权限要求

- 查看群组详情需要 `group:view`。
- 修改群组需要 `group:edit`。
- 解散群组需要 `group:delete`。
- 邀请成员需要 `group:invite`。
- 移除成员需要 `group:kick`。

### 当前限制

- 新前端工作台暂未迁移群组管理页面。
- 老组件仍在代码中，但未纳入新版显式菜单。

## 全局角色与权限管理

### 已支持

这是当前已经落地的核心全局后台能力。

支持接口：

```text
GET /api/v1/iam/roles
GET /api/v1/iam/roles/:role_id
GET /api/v1/iam/permissions
PUT /api/v1/iam/roles/:role_id/permissions
```

### 可管理内容

- 查看系统角色列表。
- 查看单个角色详情。
- 查看权限码列表。
- 全量替换某个角色绑定的权限。

### 权限要求

- 调用者必须拥有系统全局域下的 `role:manage` 权限。

### 内置角色

- `superadmin`：系统超级管理员。
- `owner`：团体/项目所有者。
- `developer`：开发者。
- `member`：普通成员。

### 内置权限码

- `app:create`：发布应用。
- `app:edit`：编辑应用。
- `app:offshelf`：下架应用。
- `app:delete`：删除应用。
- `app:subscribe`：订阅/购买应用。
- `project:create`：创建项目。
- `project:edit`：编辑项目及编排。
- `project:delete`：删除项目。
- `group:view`：查看群组。
- `group:edit`：编辑群组。
- `group:delete`：解散群组。
- `group:invite`：邀请成员。
- `group:kick`：移除成员。
- `role:manage`：管理角色与权限。

### 安全保护

- `superadmin` 不能移除 `role:manage` 权限，避免锁死权限管理入口。
- 权限变更后会清理本地 Casbin 缓存。
- 权限变更后会通过 RocketMQ 广播缓存失效消息。

### 当前限制

- 新前端还没有角色/权限管理页面。
- 只能替换角色权限，不能创建新角色或删除角色。
- 没有全局用户列表和用户角色分配页面。
- 没有可视化 Casbin policy 管理页面。

## 审计与运维能力

### 已支持

- Market 发布、上传失败、补偿删除等关键事件写入 `audit_events`。
- MinIO 删除失败会进入 `minio_delete_retries`，后台 worker 重试。
- 下载数据写入 Redis 热榜和 `app_download_metrics`。
- HTTP 服务支持优雅退出。
- 请求池过载时快速返回 `503`。
- 支持 Redis 或内存限流后端。

### 当前限制

- 还没有审计日志查询后台页面。
- 还没有 MinIO 删除补偿队列管理页面。
- 还没有限流、请求池、RocketMQ 状态可视化页面。

## 当前新版前端页面

新版前端菜单当前只开放：

```text
/market  应用市场与购买
/wallet  当前用户钱包和流水
/orders  当前用户订单
/login   登录注册
```

暂未迁移到新版菜单的能力：

- 个人资料管理。
- 群组管理。
- 角色/权限管理。
- 全局应用审核。
- 全局用户管理。
- 全局订单/钱包审计。

## 建议下一步建设的全局管理后台

建议按以下优先级补齐：

1. 全局管理员首页：系统概览、用户数、应用数、订单数、交易额、异常队列。
2. 用户管理：用户列表、启用/禁用、重置会话、查看用户群组和角色。
3. 应用审核：所有应用列表、强制下架、恢复上架、查看文件元数据。
4. 订单管理：全局订单查询、按状态过滤、查看支付流水和订阅发放。
5. 钱包审计：全局流水查询、按用户/钱包/订单过滤、导出流水。
6. 角色权限页面：角色列表、权限勾选、权限变更确认和审计。
7. 审计中心：查看 `audit_events`、MinIO 删除补偿、RocketMQ/Casbin 同步状态。

## 验收用例

### 购买应用闭环

1. 登录用户。
2. 进入 `/wallet`。
3. 如果余额不足，先充值。
4. 进入 `/market`。
5. 点击应用的“购买并发放”。
6. 页面显示 `order_id`、`tx_id`、`subscription_id`。
7. 进入 `/orders`，确认订单状态为 `paid`。
8. 回到 `/wallet`，确认出现一笔 `consume` 流水，`reference_id` 为订单 ID。

### 角色权限管理接口

1. 使用具备 `role:manage` 的管理员登录。
2. 调用 `GET /api/v1/iam/roles` 查看角色列表。
3. 调用 `GET /api/v1/iam/permissions` 查看权限列表。
4. 调用 `PUT /api/v1/iam/roles/:role_id/permissions` 替换权限。
5. 确认权限变更后对应接口访问结果发生变化。
